Overview

A personal portfolio built from scratch — no frameworks, no build tools, no dependencies. The design language draws from NASA technical documentation and early 2000s punk zine aesthetics: monospace HUD overlays, scanline effects, tactical corner brackets, and a dark void palette. The site is entirely static HTML, CSS, and vanilla JavaScript.

/// Github Repository — Portfolio
Design

Design Language

The aesthetic is inspired by Dieter Rams' ten principles of good design — particularly "as little design as possible" — reinterpreted through a NASA mission control lens. Every visual element serves a purpose: HUD brackets mark the viewport, the ticker communicates metadata, and the page-top hero doubles as an information radiator.

Typography is handled by two families: DM Sans for display and body copy (clean, humanist, legible at small sizes), and Share Tech Mono for all HUD labels, codes, and data readouts. Color palette is deliberately minimal — a near-black void, teal as the primary accent, and a burnt orange for warnings and highlights.

Engineering

How It's Built

All site-wide content (name, role, status, ping label) is driven by a single js/config.js file via a SITE_CONFIG object — a single source of truth for anything that needs to change across pages.

  • Auth gate: SHA-256 hash stored in sessionStorage — no backend required.
  • Text scramble: letter-by-letter character cycling on the nav logo, resolving to the target string.
  • Page wipe: full-viewport overlay animates in/out on navigation to smooth page transitions.
  • Skill bars: animate on scroll via IntersectionObserver, filling to a data-attribute target width.
  • Bottom ticker: CSS marquee using @keyframes scroll on a duplicated track for seamless looping.
