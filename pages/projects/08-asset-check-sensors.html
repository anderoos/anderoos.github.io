OP-008 // DATA ENGINEERING

Asset Check Sensors

LIVE 2025
Python dbt Data Quality Monitoring
Overview

Automated data quality sensors that validate asset health across pipeline stages. The system flags schema drift, null rate violations, and freshness failures before they reach downstream consumers — replacing manual spot-checks with a continuous, automated monitoring layer.

Problem

What Needed Solving

Data quality issues were being caught late — either by downstream analysts or by end users. There was no systematic layer that validated asset health after each pipeline run. Schema changes upstream would silently break models, and null rates could spike for days before anyone noticed.

  • Schema drift goes undetected until downstream models fail.
  • No automated freshness checks on time-sensitive tables.
  • Null rate spikes surface only through user complaints.
Solution

How It Was Built

Sensors were implemented as a lightweight validation layer that runs post-pipeline on each asset. Each sensor type targets a specific failure mode: schema sensors diff column signatures against a registered baseline, freshness sensors compare the latest record timestamp against an expected cadence, and null sensors measure column-level null rates against configurable thresholds.

Failures surface as structured alerts with asset name, sensor type, expected vs. observed values, and severity — giving on-call engineers enough context to triage without digging into logs.
